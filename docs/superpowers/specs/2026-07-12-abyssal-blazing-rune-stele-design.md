# 渊炽铭符碑设计

## 目标

为 SPD 添加纯装饰方块“渊炽铭符碑”。方块使用用户提供的 GeckoLib 几何模型与 128×128 贴图，占用竖直两格，支持四向放置，并在世界、物品栏和手持状态下显示完整模型。

## 已确认范围

- 方块 ID：`spd:abyssal_blazing_rune_stele`。
- 显示名：渊炽铭符碑。
- 方块为纯装饰，不提供右键交互、动画或范围效果。
- 放置时碑面朝向放置玩家，支持水平四向旋转。
- 方块占用上下两个方块空间。
- 使用 `E:\代码\MC模型\转换后\柱子.json` 作为 GeckoLib 几何来源。
- 使用 `E:\代码\MC模型\转换后\texture.png` 作为 128×128 贴图。
- 合成配方为中心 1 个紫水晶碎片，上下左右各 1 个黑曜石，产出 1 个渊炽铭符碑。

## 结构方案

采用一个带 `HALF` 与 `FACING` 状态的自定义方块，而不是两个独立方块或单格向上越界渲染。

- `HALF` 使用 `DoubleBlockHalf.LOWER` 与 `DoubleBlockHalf.UPPER` 表示上下半块。
- `FACING` 保存水平朝向，两半始终保持相同方向。
- 放置前检查目标位置与上方位置均可替换，并检查世界高度限制。
- 放置时写入下半块，再在上方写入同朝向的上半块。
- 相邻更新发现另一半缺失或状态不匹配时，残留半块自动移除。
- 玩家破坏任意一半都会移除另一半；只有下半块参与正常掉落，避免复制物品。
- 创造模式破坏上半块时不会让下半块额外掉落。

这种结构保证上半部分拥有真实占位和碰撞，玩家不能把其他方块放进模型上半部分，也不会穿过碑体。

## GeckoLib 渲染

### 世界渲染

仅下半块创建 `AbyssalBlazingRuneSteleBlockEntity`。下半块返回 `ENTITYBLOCK_ANIMATED`，上半块不创建方块实体并保持不可见占位，由下半块的 `GeoBlockRenderer` 一次绘制完整 32 像素高模型。

模型资源保存为：

- `assets/spd/geo/abyssal_blazing_rune_stele.geo.json`
- `assets/spd/textures/block/abyssal_blazing_rune_stele.png`

渲染器以方块中心为原点，并根据下半块的 `FACING` 状态绕 Y 轴旋转。模型保持原始 1:1 比例，使其 32 像素高度对应两个方块。

### 物品渲染

新增专用 `BlockItem`，同时实现 GeckoLib 物品渲染接口，复用同一份几何和贴图。Fabric 与 Forge 分别通过加载器专用客户端 Mixin 提供物品渲染器，公共模块只保留不引用客户端 API 的回退，避免再次引入 Forge 模块分包问题。

物品模型 JSON 使用 `builtin/entity`，并配置适合 GUI、地面、第一人称和第三人称的缩放与位移，确保两格高模型不会超出物品栏图标或遮挡玩家视野。

## 碰撞与属性

模型主体宽约 15 像素、深约 10 像素。上下半块各使用对应的一格高碰撞箱；南北朝向使用 15×10 的横向轮廓，东西朝向交换宽深。碰撞箱不会使用整格立方体，也不会超出模型两格高度。

方块材质按黑曜石石碑处理：

- 不透明几何由 GeckoLib 绘制，方块属性使用 `noOcclusion`。
- 使用石质音效。
- 需要正确工具才能掉落。
- 硬度与爆炸抗性沿用黑曜石基准。

## 注册与资源

- `SpdBlocks` 注册方块。
- `SpdItems` 注册专用方块物品。
- `SpdBlockEntities` 注册仅绑定下半块的方块实体类型。
- Fabric 与 Forge 客户端入口注册方块实体渲染器。
- `SpdCreativeTabs` 将方块物品加入 SPD 创造模式标签页。
- 中英文语言文件添加名称。
- blockstate 和占位 block model 覆盖全部 `facing` 与 `half` 状态，避免资源缺失警告。
- 数据包配方使用十字形 3×3 布局。
- 添加与黑曜石一致的镐挖掘标签和掉落表，保证生存模式正确掉落一个方块物品。

