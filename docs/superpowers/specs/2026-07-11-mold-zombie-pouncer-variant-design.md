# 霉染僵尸飞扑变种设计

## 目标

在现有霉染僵尸实体中加入独立的飞扑变种。该变种复用现有模型和动画，使用独立纹理，并在接近玩家时飞扑。自然生成、刷怪蛋生成及其他正常调用 `finalizeSpawn` 的生成路径均有 20% 概率生成飞扑变种。

## 状态模型

现有行走型和奔跑型状态保持不变。新增同步布尔状态 `POUNCER_VARIANT`，与现有速度变种正交，因此飞扑变种仍可能是行走型或奔跑型。

飞扑状态在服务端生成时随机一次，通过实体同步数据发送到客户端，并写入 NBT。旧存档中没有该字段的霉染僵尸默认不是飞扑变种，避免已有实体在读档后随机改变。

## 纹理

普通霉染僵尸继续使用 `textures/entity/mold_zombie.png`。飞扑变种使用 `textures/entity/mold_zombie_pouncer.png`。`MoldZombieModel#getTextureResource` 根据同步状态选择纹理；模型与动画资源路径不变。

## 飞扑行为

飞扑变种拥有专用目标行为，仅对玩家触发：

- 目标存活且距离约 3 至 6 格。
- 僵尸在地面、能看见目标且飞扑冷却结束。
- 启动时朝目标施加水平速度和向上速度，形成短距离扑跃。
- 飞扑期间检测与目标的近距离碰撞；命中时执行一次正常近战攻击。
- 成功命中玩家后附加 5 秒缓慢 I。
- 每次尝试后进入冷却，防止连续飞扑；落地或超时后结束本次尝试。

普通霉染僵尸不会获得飞扑行为。飞扑过程中不新增动画，继续使用现有移动/追逐动画。

## 兼容与验证

- 保持霉染僵尸实体 ID 和刷怪蛋 ID 不变。
- 保持现有行走型/奔跑型各自速度、静音和动画逻辑不变。
- Forge 与 Fabric 必须完成强制构建。
- Forge 成品必须包含新纹理、更新后的实体和模型类。
- 部署后的 Forge jar SHA-256 必须与构建产物一致。
