# 炽脉疾锋刀设计

## 目标

为 SPD 添加一把名为“炽脉疾锋刀”的高速近战武器。武器使用用户提供的短刀模型与纹理，通过 GeckoLib 自定义物品渲染覆盖背包、地面、第一人称和第三人称显示；战斗规则由服务端权威执行，避免客户端与服务端状态不一致。

## 已确认范围

- 物品 ID：`spd:blazing_vein_dagger`。
- 显示名：炽脉疾锋刀。
- 使用同一份 `短刀模型.json` 作为模型来源。
- 使用用户提供的短刀纹理。
- 不添加额外攻击动画，保持静态 GeckoLib 模型。
- 最终攻击伤害为 5。
- 最终攻击速度为 2.1。
- 最大耐久度为 1900。
- 物品耐热，掉入熔岩不会烧毁。
- 继承标准剑类近战附魔兼容性，包括锋利、亡灵杀手等；通过攻击入口禁止横扫伤害。
- 使用炽炭钢锭修理。
- 添加 1 个炽炭钢锭加 1 个烬柄制作 1 把短刀的基础合成配方。

## 资源与客户端渲染

### 模型

输入的 `短刀模型.json` 是 Bedrock 几何格式，包含 `minecraft:geometry`、骨骼和立方体数据，不是 Minecraft 原版 item model。实现时将其转换为 GeckoLib 4 使用的 `assets/spd/geo/blazing_vein_dagger.geo.json`，保留几何尺寸、旋转、UV 和纹理分辨率。

新增 `BlazingVeinDaggerModel`，返回短刀的 GeckoLib 几何资源和纹理资源。由于本物品没有动画，模型不依赖额外的攻击动画文件。

### 渲染器

新增 `BlazingVeinDaggerRenderer`，基于 GeckoLib 物品渲染器实现。Forge 和 Fabric 客户端初始化时分别注册该物品的渲染器。原版 item JSON 保留为资源占位和粒子纹理来源，最终物品渲染由 GeckoLib 完成。

### 资源路径

- `assets/spd/geo/blazing_vein_dagger.geo.json`
- `assets/spd/textures/item/blazing_vein_dagger.png`
- `assets/spd/models/item/blazing_vein_dagger.json`
- `assets/spd/lang/zh_cn.json`
- `assets/spd/lang/en_us.json`

## 战斗属性与状态

### 物品属性

物品继承 `SwordItem`，使用炽脉材料等级，并配置 1900 耐久和耐热属性。攻击速度属性按最终值 2.1 配置，而不是把 2.1 误当作原版攻击速度修正量。

### 疾锋

每把物品堆栈独立保存以下 NBT 状态：

- 当前层数，范围 0 到 5；
- 最后命中时间，以世界 tick 保存。

服务器处理命中时：

1. 若距上次命中超过 16 tick，先将层数清零。
2. 若命中前已经有 5 层，当前攻击触发瞬斩；否则将层数增加 1。
3. 触发瞬斩后立即清零。
4. 将当前世界 tick 写为最后命中时间。

每层疾锋使用攻击速度临时属性修正提升 4%。采用乘法叠加，速度倍率为 `1.04 ^ 层数`；5 层时约为基础速度的 1.217 倍。玩家不再持有该刀或层数归零时移除该修正。

### 瞬斩

瞬斩在已经拥有 5 层的下一次命中触发，向同一目标追加一次独立伤害判定，伤害为当前普攻伤害的 70%。第二段不会再次触发疾锋、灼脉概率或额外耐久消耗，避免一次攻击递归触发多个状态。

### 灼脉

复用现有 `spd:searing_pulse` 效果，其中文名已经是“灼脉”。刀的每次普攻以 30% 概率施加 40 tick 灼脉。效果每 20 tick 检查目标的水平移动，移动时造成 0.5 点火焰伤害。

### 疾跑闪避

当玩家疾跑并持有炽脉疾锋刀时，受到其他生物造成的直接近战伤害有 10% 概率完全取消本次伤害。投射物、环境伤害和没有直接生物攻击者的伤害不属于此判定。

### 横扫禁用

在 `Player.attack` 的横扫调用点增加定向拦截。当主手物品是炽脉疾锋刀时跳过横扫攻击，仍保留剑类附魔兼容性。

## 代码边界

- `SpdItems`：注册物品。
- `SpdCreativeTabs`：加入 SPD 创造模式标签页。
- `BlazingVeinDaggerItem`：物品属性、疾锋 NBT、瞬斩、灼脉、耐久和修理规则。
- `PlayerMixin`：横扫拦截和疾跑闪避入口。
- GeckoLib model/renderer：只在客户端初始化阶段使用。
- 数据资源：物品模型、纹理、语言和合成配方。

服务端逻辑不直接引用客户端类。状态使用物品堆栈 NBT，保证丢弃、切换和重新登录后行为可恢复，并避免玩家之间共享层数。

## 验证

添加针对纯状态计算的单元测试，覆盖超时清零、层数上限、满层下一击触发、70% 第二段伤害和速度倍率。资源测试检查物品注册使用的资源文件、语言键和配方存在。

实现完成后至少运行：

```text
gradle :common:test --console=plain
gradle build --console=plain
```

同时检查 Forge 与 Fabric 产物都能编译，客户端日志中没有 GeckoLib 资源缺失或客户端类被服务端加载的错误。
